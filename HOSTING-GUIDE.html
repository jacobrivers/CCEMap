OverviewHow It Works

Context: NiCE Expert (formerly MindTouch) is a knowledge management platform — it is not a web server. The map must be hosted externally. This guide uses GitHub Pages (free) as the host, then embeds the map into a NiCE Expert page using an iframe. No coding is required beyond pasting configuration values.
1. GitHub Repository Stores the map files (HTML, CSS, JS, JSON data)
→
2. GitHub Pages Serves the map as a live website at a public URL
→
3. NiCE Expert Page Embeds the map via iframe — users see it inline

SetupWhat's in This Folder

Upload the entire NiCE CXone Map - MindTouch folder to GitHub. Its structure:

nice-cxone-map/ ├── index.html ← Main map page ├── config.json ← Edit this to switch to GitHub data mode ├── css/ │ └── map.css ├── js/ │ └── map.js └── data/ ├── locations.json ← All map pins └── platforms.json ← Platform/service types

Part 1Host on GitHub Pages

Estimated time: 10–15 minutes. You need a free GitHub account.

Step 1 — Create a GitHub account (skip if you have one)

1

Go to github.com and click Sign up.

2

Choose a username (this will appear in your map URL, e.g. myname.github.io), enter your email and a password, then complete the verification.

Step 2 — Create a new repository

1

Once logged in, click the + icon in the top-right corner and select New repository.

2

Fill in the form:
— Repository name: nice-cxone-map (or any name you prefer)
— Description: Optional — e.g. "NiCE CXone Global Map"
— Visibility: Select Public

3

Leave all other options at their defaults. Click Create repository.

📌 Note: Public is required for free GitHub Pages hosting. Private repos require a paid GitHub plan.

Step 3 — Upload the map files

1

On your new repository page, click uploading an existing file (or go to Add file → Upload files).

2

Drag and drop (or browse to select) all files from the NiCE CXone Map - MindTouch folder. You need:

index.html
config.json
css/ (folder)
js/ (folder)
data/ (folder)

⚠️ GitHub's web uploader doesn't accept folders directly. Drag the contents of each folder into the correct path, or use the GitHub Desktop app (free) which handles folders automatically.

3

Scroll down to the Commit changes section. Leave the default message and click Commit changes.

4

Alternative — GitHub Desktop (easier for folders):

Download GitHub Desktop → Clone your new repo → Drag the map folder contents into the cloned folder → Click Commit to main → Click Push origin.

Step 4 — Enable GitHub Pages

1

In your repository, click the Settings tab (gear icon, top-right of the repo).

2

In the left sidebar, click Pages.

3

Under Branch, select main from the dropdown and leave the folder as / (root). Click Save.

4

Wait 2–5 minutes. Refresh the page. A green banner will appear with your map URL:
https://YOUR-USERNAME.github.io/nice-cxone-map/
Copy this URL — you need it for Part 2.

5

Open the URL in a browser to confirm the map loads correctly.

Step 5 (optional) — Switch to GitHub data mode Optional

By default the map reads data from the local data/ folder you uploaded. Switching to GitHub data mode lets the map always fetch the latest data directly from your repo — so you can update pins by editing files in GitHub without re-uploading everything.

1

Open config.json in your repository (click the file, then the pencil ✏️ icon to edit).

2

Change "dataSource" to "github" and set githubRawBase to your raw GitHub data URL:

{
  "dataSource": "github",
  "githubRawBase": "https://raw.githubusercontent.com/YOUR-USERNAME/nice-cxone-map/main/data/",
  "mapTitle": "Global AI CX Reach & Coverage",
  "defaultTheme": "dark",
  "defaultLabels": true
}

Replace YOUR-USERNAME with your actual GitHub username.

3

Scroll down and click Commit changes. The map will now load location data live from GitHub.

Part 2Embed in NiCE Expert

Prerequisites: Complete Part 1 first. You need your GitHub Pages URL and access to edit pages in NiCE Expert. Confirm with your Expert administrator that iframe embeds are permitted in your instance.

Option A — iframe embed (recommended)

This places the map as a live interactive widget inside any Expert page.

1

Open the Expert page you want to embed the map in and click Edit.

2

Switch to the Source / HTML view of the editor (usually a <> or HTML button in the toolbar).

3

Paste the following iframe code where you want the map to appear:

<iframe src="https://YOUR-USERNAME.github.io/nice-cxone-map/" width="100%" height="700" frameborder="0" allow="fullscreen" style="border:none;border-radius:8px;" title="NiCE CXone Global Map"> </iframe>

Replace YOUR-USERNAME with your GitHub username and nice-cxone-map with your repo name.

4

Adjust height="700" to suit your page layout. Values between 600–800 work well.

5

Click Save. The map will appear as an interactive embedded widget on the page.

Option B — Direct link (simpler)

Instead of embedding, link directly to the map URL. The map opens in a new tab rather than appearing inline on the Expert page.

1

In Expert, highlight the text you want to turn into a link (e.g. "View Interactive Map").

2

Click the link icon in the editor toolbar and paste your GitHub Pages URL.

3

Save the page. Users click the link to open the map in a new tab.

Troubleshooting

SymptomFix
Blank iframe / security error in Expert Your Expert instance may require the GitHub Pages domain to be allowlisted. Contact your Expert admin and provide the domain (e.g. YOUR-USERNAME.github.io) to add as a trusted iframe source.
Map loads in browser but not in Expert iframe Some Expert configurations block third-party iframes by default. Request that your admin add an exception for your GitHub Pages domain in the Content Security Policy settings.
GitHub Pages URL returns 404 Wait 5 minutes after enabling Pages and try again. If still failing, check Settings → Pages to confirm GitHub Pages is enabled and the branch is set to main.
"Could not load data from GitHub" on map Check config.json: githubRawBase must exactly match your username and repo name. The data/ folder must contain both locations.json and platforms.json.
Map pins disappeared after update The browser may have cached old data. Click 🔄 Sync in the map controls to reload from source. Or force-refresh the browser (Ctrl+Shift+R / Cmd+Shift+R).

Part 3Managing Map Data

When you need to add, edit, or remove locations — one person makes changes and publishes; all Expert readers see the update on their next page refresh.

Data update workflow (GitHub data mode)

1

Open the live map at your GitHub Pages URL. Make your changes — add, edit, or remove locations using the map controls.

2

Click 💾 Export in the map controls. This downloads nice-cxone-map-settings.json — a single file containing all locations, platforms, colours, and theme settings.

3

Open your GitHub repository. Navigate into the data/ folder. Click Add file → Upload files.

If you want to update individual data files instead, you can also extract and upload just locations.json and platforms.json — but the settings export is the easiest single-file approach.

4

Replace the existing files with your updated versions. Scroll down and click Commit changes.

5

GitHub Pages rebuilds automatically (typically within 1–2 minutes). All users who refresh their Expert page will see the updated map.

6

Use the 🔄 Sync button on the map to reload from GitHub and confirm your changes published correctly.

💡 Tip: The 🔄 Sync button discards the browser's local working copy and reloads from the published GitHub files. Use it to confirm your upload worked, or to reset after experimenting.

Export / Import settings

✓ New feature — The map now has full settings export/import, including marker type colours and labels.
ActionWhat it does
💾 Export Downloads a single nice-cxone-map-settings.json file containing all locations, platforms, marker type config (colours + labels), theme (dark/light), and label visibility. You can open and edit this file in any text editor.
📂 Import Picks a nice-cxone-map-settings.json file and applies everything — locations, platforms, colours, theme — live to the map. Use this to share a full configuration or restore a saved state.
🎨 Marker Types Opens a modal to change the hex colour and label text for each of the three marker types (CXone, Voice POP, SOV). Changes update all pins, layer buttons, and the legend instantly.
🔄 Reset Resets all locations, platforms, colours, and theme back to factory defaults. Clears the browser's localStorage.

config.json reference

{
  "dataSource": "local",        // "local" = use data/ folder | "github" = fetch from GitHub
  "githubRawBase": "https://raw.githubusercontent.com/YOUR-USERNAME/YOUR-REPO/main/data/",
  "mapTitle": "Global AI CX Reach & Coverage",
  "defaultTheme": "dark",       // "dark" or "light"
  "defaultLabels": true         // true = map labels on by default
}

Note: defaultTheme and defaultLabels in config.json are the initial defaults. Once a user changes them, their choice is saved to localStorage and takes precedence.
